Regular expressions have long been the bane of me. Still, they’re very useful for front end validation and can even be used on the server side for secondary validation. I’ll have to learn how to use them and learn to enjoy it too.. As per usual, the preferred way to create a regular expression (as with almost any object) is to use a literal. Groups of characters are tracked by placing them inside of a set of square brackets. For example, [aeiou] will track vowels and [qwstr] will check if a latter in that section of the regexp is either q, w, s, t or r. Sequences can be tracked using a dash “-“. Regexp are case sensitive by default and can be made case insensitive by appending “i” to the end of the regexp. This is because regexp are objects and have built in methods. “g” is also a method and it returns all matching cases, not just the first one. “m” allows the regexp to read multiple lines. An example of using one of the methods is as follows, “pattern = /java/i”.

Metacharacters as defined in the readings (JavaScript novice to ninja 2nd edition)

  • . matches any character, except line breaks
  • \w matches any word character, and is equivalent to [A-Za-z0-9_]
  • \W matches any non-word character, and is equivalent to [\^A-Za-z0-9_]
  • \d matches any digit character, and is equivalent to [0-9]
  • \D matches any non-digit character, and is equivalent to [^0-9]
  • \s matches any whitespace character, and is equivalent to [ \t\r\n\f]
  • \S matches any non-whitespace character, and is equivalent to [^ \t\r\n\f]

Modifiers as defined in the readings (JavaScript novice to ninja 2nd edition)

  • ? makes the preceding token in the regular expression optional
  • * matches one or more occurrences of the preceding token
  • + matches one or more occurrences of the preceding token
  • {n} matchesnoccurrences of the preceding token
  • {n,} matches at leastnoccurrences of the pattern
  • {,m} matches at mostmoccurrences of the preceding token
  • {n,m} matches at leastnand at mostmoccurrences of the preceding token
  • ^ marks the position immediately before the first character in the string
  • $ marks the position immediately after the last character in the string

Special characters and modifiers need to be escaped using a backslash. Without a backslash, they can not be matched in a regexp.

The following regexp would match any word that starts with a capital letter and dosen't have numbers in it (I think). "^[A-Z][a-z]*\D$". This is the case when tested on https://regex101.com/. This is tested in the activites under part one. and found in the soucres tab under part1.js.

Capturing and Bubbling. In a nutshell, bubbling goes from the target event and rides all the back “up” to the document root. This will fire events going up along the way. Capturing goes in the opposite direction and goes “down” the dom tree until it hits the target event, firing off events from parent elements along the way. Because both can be useful, a third argument exists in addEventListener which is a Boolean value. If this is set to true, the event will be handled by capturing, if it is false, it will be handled by bubbling. Bubbling is the default behavior. Events can both capture and bubble, but you need to use the method twice to achieve this.
