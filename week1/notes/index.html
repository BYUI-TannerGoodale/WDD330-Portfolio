Pre-reading Notes

Localstorage seems like it would be usefull for reducing the load on servers an databases, at least for user centric temporary data like contents of a shoppig cart or somthing along those lines. I can see that it's no replacement for a database though, as a database would hold infrmation that was more in line with business or server needs that would need to be kept indefinitely.

Questions to answer

  1. How can I store data so it isnâ€™t lost when the application stops?
    • A database can store various inputs from a user and can also aid in delivering outputs.
    • Localstorage can save data on a client device much like a cookie, but that won't be deleted when the website instace is ended. It uses name-value pairs like a session or cookie as well.
  2. How do I get data back into my application once it has been stored?
    • JavaScript has built in methods that can call data saved on localStorage. You only need define where that data goes.
    • An API call to a database or a server connected to one can do similar things, but with data that is stored off the client and is not always in a key-value pair.
  3. What can I do to allow the user to communicate?
    • Giving the user input options through some form is the standard way to set up communication.
    • Event listeners can also be used to get ques from a user.
